First Day at the Game Studio

A client-side metagame prototype: talk with an AI-shaped team, collect ideas, ask the programmer to turn those ideas into transparent behaviour cards, then attach behaviours to sprites and playtest. Optional live AI can be enabled from the Live AI setup button.

How to play this prototype

First project: make a simple astronaut rescue game. Start with the boss, then consult teammates. Specialists provide ideas; the programmer turns those ideas into behaviour cards.

Second project: after a safe rescue, ask the boss to start Lunar Lander. The path is explicit: scientist for vertical motion and gravity, designer for safe touchdown, programmer for behaviour cards.

Behaviour cards are deliberately transparent. You can attach even the “wrong” card to an object. The prototype lets mistakes happen because they are useful material for discussion and revision.

Optional live AI: use the gold Live AI setup button to enter your own OpenAI, Gemini, or Claude API key. Live answers are advisory; the scripted buttons still control resource unlocks so the learning path remains stable.

Live AI teammate settings

Use your own provider key for free-form questions to the boss or any teammate. Keys are never included in the project JSON export. If you enable caching, the key is stored in this browser's localStorage.

Browser-only API calls are convenient for prototyping, but they are not production security. Anyone with access to this browser profile can potentially read cached keys. For a deployed classroom version, use a small server-side proxy instead.
