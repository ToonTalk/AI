A Guide for Teachers

The Atelier is a single-file web app (no install, no server, no account) in which students build small constructionist agents: little software minds that make turtle drawings or short poems/stories, exhibit them, receive critique, reflect in a journal, and improve over rounds. An optional Science Corner lets older students run controlled experiments on those minds.

It runs in any modern browser — from a file, a static server, or inside a Claude artifact. All data stays in the student's browser unless they export it.

Why it's built this way (the pedagogy)

The app is an object-to-think-with at two levels, after Seymour Papert:

  1. The agent's artifacts (drawings, poems, stories) are public, shareable constructions — classic constructionism.
  2. The agent's mind itself is a constructed, editable object. When an agent stagnates, loops, or improves, the student debugs its learning process — its goals, skills, self-perception, and memory. This lifts Papert's "debugging as the heart of learning" up one level.

Design pillars worth naming for your students:

  • Glass box, never black box. Every part of a mind is plain text the student can read and rewrite. Every AI call exposes a "peek under the hood" view of the exact prompt and reply. Nothing is hidden.
  • Make → exhibit → critique → reflect → improve — for the agent and the student (dual reflection; see the Notebook).
  • Society of Mind, in plain words. After Marvin Minsky: Maker, Eyes, Critic, and Memory are little agents in a tiny society. Small optional "Big Idea" bubbles connect features to Papert and Minsky in child-readable language.
  • Amplify, don't substitute. The student is the director and editor. The app repeatedly hands agency back: edit the artifact, edit the mind, curate the memory, decide what "better" means.

Setup (5 minutes)

Open Settings and choose how the AI helper is reached:

  • Pretend helper (mock mode). No internet, no keys, fully deterministic. Start here. The entire loop — including experiments — runs offline with canned responses, so students can explore mechanics before any real calls. Toggle it on in Settings (or add ?mock=1 to the URL).
  • Artifact mode (keyless). Inside a Claude artifact, the helper works without a key. Note: a Claude artifact cannot save between visits — a banner warns students, and they should Settings → Export to keep a backup.
  • Standalone (bring your own key). Add an Anthropic, OpenAI, or Google API key. Keys are stored only in the browser, never in exports, and only persist if the student opts in. You set a per-call token budget and the model used for each job.

Other switches:

  • Advanced mode unlocks the Science Corner and opens the "peek under the hood" panels by default. Turn it on for older students; leave it off for the youngest.
  • A small cost meter shows calls made and a rough estimate. The Science Corner shows an estimated call count before a run.
Tip: for a first lesson with no accounts and no cost, just use mock mode. It's designed for exactly this.

A first lesson (≈ 45 min, mock mode)

  1. Make a mind from a starter template (e.g. Spiral Gardener). 5 min.
  2. Run 3 rounds. Watch the phases fill in; replay a drawing. Ask: what changed, and why? 10 min.
  3. Open its mind. Tour the parts — Spark, Hands, Eyes, Memory. Change one thing. Run a round. 10 min.
  4. Edit a lesson in Memory (delete or add one). Run a round and find the difference in the journal. 10 min.
  5. Notebook reflection on the prompt the app offers. 10 min.

Extension: make a second mind and hold a Show & Tell.

The student's loop, at a glance

A round runs seven inspectable phases: Make → Render → Look → Exhibit → Critique → Reflect → Distill (Distill every 3rd round, compressing the journal into ≤ 5 lessons). Lessons are the agent's only long-term memory fed back into Make, and they are fully student-editable — the most important control surface in the app.

Show & Tell lets 2–4 minds present their newest work and give each other kind notes. The notes genuinely feed back into each author's next round, and a 📌 badge shows when a mind has notes waiting. Hard-coded kindness rules govern every critique: name something specific that works, offer one wish, never mock, talk about the work not the maker.

The Science Corner (older students)

A controlled-experiment wizard whose hidden curriculum is operationalizing "better" and meeting Goodhart's law honestly.

  • Clone a base mind into baseline + variant, change exactly one part, run N rounds.
  • A fixed judge (same prompt and model for the whole experiment) scores each work; you also get the agent's self-rating and a novelty metric.
  • Results show as paired timeline strips and hand-rolled SVG charts, exportable as CSV (long format) for spreadsheet work.
  • Built-in discussion prompts push the key question: did the agent get better at art, or just better at pleasing the judge?

Great cross-curricular hooks: fair tests and variables (science), measurement and graphs (maths), what counts as "good" art or writing (English/art), and the ethics of optimisation (PSHE / computing).

Assessment ideas

  • The Notebook is the student's own journal (separate from the agents'), with rotating reflective prompts and Markdown export — a ready-made portfolio of metacognition.
  • Have students export a mind's rounds as a web page and annotate what changed and why.
  • In the Science Corner, mark the quality of the question and the interpretation, not the chart shape — especially whether the student notices their metric being gamed.
  • Look for evidence of debugging the mind: a clear hypothesis, one change, and a reasoned conclusion.

Safety, privacy, and tone

  • No personal data is ever requested. Agent names are fictional; the app never asks for student names or emails.
  • Every AI call carries a fixed, visible safety preamble: the audience is children; no violence, romance, scary or mature themes; warm, simple language. Model output is also stripped of markdown and sanitised before storage.
  • Kindness is enforced in every critique and judge call (specific praise, one gentle wish, never mock or compare people).
  • Keys and storage: API keys never leave the browser and are never exported. Warn students that any file they export may be shared, so they should only share with people they trust. Imported trading cards and backups are sanitised on the way in.

Running it without the internet or a server

  • Works from a double-clicked file or a static server. On a fresh non-embedded start it defaults to standalone + mock so the first action just works.
  • Exports: trading cards (one mind), gallery pages, a mind's rounds, the notebook (Markdown), and a full JSON backup. Inside an artifact, downloads fall back to a copy-and-paste box.
  • For a class wall, have each student export their gallery or best mind and collect them.

Quick troubleshooting

SymptomLikely cause / fix
"Could not find that model"Standalone mode with a wrong/old model id in Settings. Switch provider (defaults reset) or type a current id.
"The helper didn't answer / busy"Rate limit or usage cap. Wait a minute and press Try this round again; completed phases are preserved.
Nothing saves between visitsYou're inside an artifact (a banner says so). Use Settings → Export for a backup.
Download buttons do nothingSame — an artifact blocks downloads; a copy-and-paste box appears instead.

A few discussion questions to keep in your pocket

  • When you edit what your agent made, whose work is it?
  • Your agent kept making the same thing. What do you do when you're stuck?
  • What does "better" mean for this mind — and who gets to decide?
  • If you took away its memory, what would change, and how would that feel?
