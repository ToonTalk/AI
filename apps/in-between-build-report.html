03

What it became

All of it in one HTML file — no build step, no framework, no packages. The only optional extras are web fonts and, for the AI features, a language-model endpoint.

Two-way morphing

Interpolate between two turtle drawings — and extrapolate past either end into the "beyond" zone, now with tabs that make it discoverable.

A 3D beetle

The same idea in space, with three ways to blend rotation: axis-angle, slerp, and yaw/pitch/roll.

A real Logo interpreter

Procedures, parameters, if/stop, recursion, and arithmetic — fractals included.

Perceptual colour

Colours blend in OKLab so the in-between hues look even — and they extrapolate too.

Three AI minds

Describe a shape in words and a model writes the turtle code. Claude runs keyless inside the artifact; Gemini and ChatGPT work with your own key.

Tune anything

A slider for every number in a program and a colour picker for every pen colour, live.

A grounded tutor

Ask why a blend looks the way it does; the answer is built from this blend's actual numbers, not guesswork.

Closure-aware by default

The opening pair — a five-point star into a decagon — is chosen so every in-between is a closed figure. The maths picks what a child sees first.

Shareable exports

Save any morph as its own self-contained, still-interactive HTML file — the engine and the AI ride along inside it.

A way home

Every exported file carries a link back to the live app, and the link updates as you scrub so it reopens exactly the frame on screen.

Live, shareable URLs

The app writes your current morph into the address bar as you work, so any state is a copy-and-paste away from being shared.

Zero dependencies

One file, vanilla JavaScript, no build and no packages. Email it, host it, or open it offline — it just runs.

04

Things we worked out

The parts that were more than typing — small results that fell out of building carefully and asking why.

A closure theorem, by gcd

Two regular polygons morph through closed in-between figures exactly when their side-counts share a factor: gcd(n, m) ≥ 2. A shared factor k gives the blend k-fold rotational symmetry, and a symmetric figure must close. Coprime pairs (triangle↔square) leave a gap — and we can predict its size. Verified for every pair from 3 to 12 sides.

The default is that theorem, applied

The first thing the app shows isn't arbitrary. A five-point star and a decagon share five-fold symmetry, so gcd(5, 10) = 5 guarantees every blend between them is a closed figure. The opening screen is the result above, turned into a design decision.

Recursion came for free

Expanding a recursive program down to flat turtle steps means the existing alignment machinery — written for plain repeat — already generalises Ken's depth-matching trick across different recursion depths, with no new blending code at all.

Closing ≠ same heading

A figure can finish facing its start direction (total turning = 360°) yet not return to its start point. Closure is a condition on the sum of displacement vectors, not on heading — which is exactly why an un-anchored intrinsic morph appears to "swim" as you slide it.

The correspondence wall

Morphing two structurally different solids (a cube into a pyramid) runs into the same hardness as coprime polygons: their per-step rotation axes are antipodal, so no honest step-for-step alignment exists. Some morphs are clean; some are genuinely open problems. We named it rather than faking it.

AI that survives being shared

The generator and the tutor call Claude with no key when they run inside the artifact, and fall back to a user-supplied Gemini or ChatGPT key anywhere else. The intelligence ships inside the one HTML file — email it to someone and the AI still works.
