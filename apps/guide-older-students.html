A Guide for Older Students

The Atelier is a place to build small constructionist agents — little software minds that make things (turtle drawings or short poems/stories), look at their own work, take critique, reflect, and improve over rounds.

Here's the twist: the agent's mind is itself an object you can read and rewrite. When an agent gets stuck, loops, or improves, you're not just editing its art — you're debugging how it learns. That's the real game.

Two ideas to keep in mind:

  • Glass box, never black box. Nothing about a mind is hidden. Every instruction is plain text, and every call to the AI has a "peek under the hood" panel showing the exact words sent and the exact reply.
  • Make → exhibit → critique → reflect → improve. This loop runs for the agent and for you.

1. The anatomy of a mind

Open any mind with Edit its mind. It has these parts, all editable:

PartWhat it controls
SparkWhat it longs to make — its goal, in its own words.
HandsThe instructions it follows each round to make something.
EyesHow it examines its own work. Can be switched off, and can remember the previous work to compare against.
Inner VoiceHow it reflects after making something.
Studio MannerHow it titles/presents its work, and the persona it uses when critiquing others.
MemoryIts lessons (long-term) and its journal (per round).

Writers can switch between poems and prose stories with the Writes picker.

Try: change exactly one part and run 3 rounds. Can you predict the effect before you look?

2. A round, phase by phase

Each round runs seven phases, and you can inspect every one:

  1. Make — the agent writes turtle code or a poem/story, plus a one-line intention. Its prompt includes its Spark, active lessons, last journal entry, last work, and any unread critiques.
  2. Render — turtle code runs in a sandboxed iframe (it can't touch the rest of the page) and animates on the canvas; writers render as a typographic card.
  3. Look — if Eyes are on, it compares this work to the last one and notes what changed.
  4. Exhibit — the work goes on the Gallery Wall.
  5. Critique — notes attach to the work (yours, or other agents' from Show & Tell).
  6. Reflect — it writes a journal entry and rates itself 1–5.
  7. Distill — every 3rd round it squeezes its journal into ≤ 5 lessons. You'll see exactly which lessons were added, kept, or dropped.

Lessons are the agent's only long-term memory injected into Make. You can edit, pin, delete, or add your own — this is the single most powerful control you have. Use it deliberately.

3. Debugging a mind

When a mind stagnates or loops, treat it like a bug hunt:

  • Is its Spark vague? Sharpen it.
  • Are its Hands telling it to do too much, or nothing new?
  • Are its lessons leading it astray? Delete the bad one.
  • What happens with Eyes off for a few rounds — does the work drift from the Spark?
  • Read peek under the hood to see what it actually received. Often the bug is in the words, not the AI.

4. Show & Tell (real critique)

Pick 2–4 minds that have each made a work. Each presents its newest piece; the others respond kindly (one thing that works, one wish). Crucially, the notes feed back: the next time a critiqued mind makes something, it reads the notes about its work — and if a critic has seen that author before, it remembers its own earlier note and can notice growth.

You can join in as a moderator, and the whole transcript is saved.

5. The Science Corner (Advanced mode)

Turn on Advanced mode in Settings to unlock experiments. This is where it gets rigorous.

  1. Pick a base mind. The app clones it twice: baseline and variant.
  2. Change exactly one part on the variant (the wizard enforces one change — toggles count).
  3. Set the number of rounds and write your question/hypothesis.
  4. A fixed judge (same prompt and same model for the whole run) scores each work 1–5 against the Spark.

You get three metrics per round, charted side by side:

  • Self-rating — the agent's own score.
  • Judge rating — the fixed judge's score.
  • Novelty — how different each work is from the one before (image hash for drawings, word overlap for writing).

Then comes the real lesson. The discussion prompts ask things like: "Your judge measured X. Did the agent get better at art, or better at pleasing the judge?" This is Goodhart's law: when a measure becomes a target, it stops being a good measure. Operationalizing "better" — and noticing when your metric is gaming you — is the hidden curriculum here.

Export the results as CSV to analyse in a spreadsheet. If a run fails partway, you can Resume it.

6. Sharing and keeping your work

  • Save these rounds as a web page — a self-contained HTML record of a mind's rounds.
  • Export this wall as HTML — a gallery page.
  • Trading card — export one mind (its anatomy, lessons, journal, best works) as JSON. Because drawings are reproducible from a code + a random seed, the person who imports your card regenerates the exact same drawings.
  • Full backup / import — everything as one JSON file.
Everything lives only in this browser unless you export it. If you're running inside a Claude artifact, downloads may pop up a copy-and-paste box instead — save the text as a file.

7. Questions worth chasing

  • Does an agent that can perceive its own trajectory (Eyes that remember) improve faster than one that can't?
  • Whose work is it when you edit what your agent made?
  • If you ran the same experiment again, would you get the same chart? What does that tell you about your measurement?
