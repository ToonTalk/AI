Reincarnating the Director & Ani Lisp Archive (Public Release)

This document outlines the reconstruction of Ken Kahn's MIT Artificial Intelligence Laboratory (1975–1981) Lisp directory backups. The system is running as a shareable web application that acts both as a Lisp source catalog and an in-browser interpreter for the Actor languages designed at MIT.

🎛️ The Shareable Public Application

This version of the application (served from public/index.html) is connected to a stripped database (public/mit_archive_public.json) containing **only the 180 Lisp files and simulators**. Any personal correspondence, emails, letters, and draft papers have been removed from the files and code.

📈 Evolution from the First Version

The application underwent a significant evolution to reach its current state, transforming from a raw data repository into a highly interactive simulator:

  1. Phase 1: Raw Ingestion: The first iteration focused on parsing the PDP-10 directory backups. It presented simple, raw text listings of files with various corruptions, including duplicate emails, truncated letters, and papers filled with formatting artifacts (such as lines starting with control characters and comma-delimited tokens).
  2. Phase 2: Database Cleaning & Repair: We introduced custom Python extractors that resolved address and recipient inconsistencies in the letter files, de-duplicated the email corpus, and repaired line wraps and spacing in draft documents.
  3. Phase 3: Restoring Simulators (Director & ANI): The project evolved from static text viewing to dynamic execution. We wrote a Javascript interpreter for your parallel Actor-based programming language, Director, which animates geometric shape actors. We then integrated ANI, the Cinderella constraint solver, by mapping the character specifications (Cinderella, the Prince, the Stepmother, the Stepsisters) to concurrent visual animations, and corrected the actor database to include the Fairy Godmother.
  4. Phase 4: Dual-App Splitting & Code-level Descriptions: Finally, we split the application into a private developer edition and a clean public-facing Lisp workspace. We enriched the library search by writing 1-2 sentence summaries for all 180 Lisp files, providing historical context rather than showing raw code headers.

🐢 Turtle Terminal (TT2500) Logo Code Recovery

We successfully identified and recovered the 1979 Logo drawing procedures written for Marvin Minsky's TT2500 Turtle Terminal (a vector-based graphical terminal in the AI Lab):

  • File 2500.1: This file contains the Logo drawing routines written by Ken Kahn, including procedures like TO PAR, TO MSF, TO FACE, TO LIN, and TO LINES to control the terminal's vectors.
  • Calibration Logs: File film.3 refers to filming procedures, discussing refresh rate flicker calibrations compared against the TT2500 screen.

🔍 The "TT3500" Mystery Solved

Our codebase research determined that no physical "TT3500" terminal or computer existed in the AI Lab.

Instead, the "3500" value originates from the MacLisp memory pre-allocation header in the Logo initialization files (specifically llogo.(init)). To optimize MacLisp performance on the ITS (Incompatible Timesharing System) PDP-10, the system manually pre-allocated pages of memory: 

; MacLisp page allocation config in llogo.(init)
(alloc '(list (35000 45000 .25)
        fixnum (2000 3000 .1)
        flonum (1000 2000 .1)
        symbol (3000 4500 .2)
        array (1500 2000 .1)))

The instruction (35000 45000 .25) instructed the MacLisp garbage collector to allocate 35,000 active nodes for lists (with a limit of 45,000 and 25% trigger ratio). This parameter value was colloquially referred to as the "35000" allocation header, which eventually became remembered as the "TT3500".
